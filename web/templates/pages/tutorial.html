API 地址格式

Orchids/Warp/Bolt/Puter: /{渠道}/v1
Grok: /grok/v1
⚠️
Claude Code 用户注意: 配置 API 地址时请使用 /{渠道}， 不要添加 /v1 后缀
Grok 路由说明: Grok 与其他渠道保持一致，使用 /grok/v1 前缀。
Bolt 账号说明: Bolt 账号需要同时填写 __session 和 project_id。
Puter 账号说明: Puter 账号当前只需要填写 auth_token。
🧭
Bolt 模型说明: 目前文档和模型管理里的 Bolt 模型采用“保守收录”策略，只放入前端主模型切换器里明确出现、且我们已在 bundle 中确认的模型： claude-sonnet-4-6、 claude-opus-4-6、 claude-opus-4-5-20251101。
前端配置里还会出现实验默认值或活动文案，但这些信息不等于“稳定公开可选模型”，所以不会直接并入公开模型列表。
Puter 模型说明: Puter 当前采用参考仓库里的“无前缀主模型”清单，已包含 Claude / OpenAI / Gemini / Grok / DeepSeek / Mistral 等主模型族； 但像 openrouter:、 togetherai: 这类聚合源前缀模型暂时不直接暴露到模型管理里。
